3. Enunt

Se dorește realizarea unei aplicații web prin intermediul căreia să se realizeze managementul fotografiilor personale. Aplicația va permite preluarea (upload-ul) de fotografii, ștergerea, organizarea în albume și partajarea acestora în mod public. In funcție de diversele tag-uri asociate unei fotografii, se va putea permite filtrarea acestora după diverse criterii (e.g. fotografiile din ultima luna, toate fotografiile în care apare o mașină roșie lângă un copac etc.). De asemenea, vor putea fi identificate/adnotate diverse obiecte sau persoane prezente intr-o fotografie – actiune ce va putea fi realizata de către orice utilizator, dar aprobata doar de posesorul fotografiei. Pentru orice fotografie, utilizatorii vor putea adăuga comentarii, acestea putand fi redate automat sau daca au fost aprobate de posesorului fotografiei. Din aplicație se vor putea exporta în format CSV diverse statistici (precum numărul de fotografii pe care un utilizator le are, câte dintre acestea au un anumit tag si altele). In plus, se va oferi posibilitatea de editare a fotografiilor la nivel minimal prin aplicarea de diverse filtre și efecte disponibile în CSS (efecte ce vor fi stocate în baza de date și aplicate fotografiei ori de câte ori va fi afișată).

5.3.1. Signup/Login Process

Signup

Client Side

On the signup.php page:

  • The user is able to easly signup with an username, email and password. Said user will be asked to confirm the password by writing it again such that the server will check that the passwords are the same.
  • There's also the option to sigup using a Google account.
Server Side

After fetching the data from the client, the server will connect to a MySQL database in order to add the data to the 'users' table in server.php.
If there is an existing user with the same username or email, then the client will be informed that the account cannot be created with the data provided. Otherwise, the account will be added to the database (with an auto incremented id and no description), and the user will be redirected to the Main page, index.php.

5.3.2. Main Page

In index.php, which can be accessed only with an account:

  • This is the page where the user is redirected after creating/accessing the account.
  • Provides a search bar which is meant to redirect the user to the filterPage.php with the tags entered in the text field.
  • Displays a logout button, meant to be used when the user wants to exit their account.

5.3.3. Search Tags

When entering words in the search bar on the index.php and clicking on the search icon the user is redirected to the filterPage.php.
The link will also contain the words provided by the user. The server extracts the words and invokes the get_tagPhotos() function from the fetchFilters.php file.
In fetchFilters.php:

  • The server will get all the public photos from the database in order to access the tags list and edits list (for displaying with the css filters).
  • When a public photo contains all the tags requested by the client, the server will add it to a list which will be sent to the filterPage.php to be displayed. The list contains ImageInfo objects (the class which stores the info about a photo extracted from the db).
  • The photo must contain all the tags requested, but can have additional tags.
  • If no photos are found, a message will be displayed to inform the user.
  • The user can access the photo.php(see Photos) file for each image that will be displayed.

5.3.4. User

The user.php page can be accessed:

  • From the navigation bar, visible to any registered user from almost all pages.
  • From any public photo the user owns. In this case, the user will access another user's page which means that the information displayed will consist only of the username, user description and the user's public photos. The link to the user.php page will contain guest parameter to inform the server that the page will be viewed in guest mode.

The user.php page for the session user (if the user views its own page):

  • All the photos owned by the user will be displayed, but the user can choose to see only the private/public photos by hovering over the Visibility button and picking an option.
  • The user can edit the description by clicking the Edit description button located under the description.
    • Also, if the user didn't set a description yet, No description will be displayed instead.
    • After clicking the button, a pop up form will apear in which the user can set the description.
  • A Upload button is provided through which the user will be redirected to the uploadPage.php in order to upload a new photo.

UserInfo.php class:

  • Contains:
    • the user id;
    • username;
    • email;
    • description;
  • Purpose:
    • To easly store the information about a user extracted from the database;
    • Used when constructing the user.php page.

5.3.5. Photos

ImageInfo.php class:

  • Contains:
    • the photo id;
    • owner;
    • filename;
    • created (the date at which the photo was uploaded);
    • visibility (public or private(default));
    • albums (the albums the photo was added to);
    • tags (the list of tags added to the photo);
    • edits (the list of css filters added to the photo).
  • Purpose:
    • To easly store the information about a photo extracted from the database;
    • Used almost everytime a photo is displayed.

The photo.php page:

  • When displayed for guest:
    • Information about the photo is displayed: the owner, the date when it was added, the visibility (public) and the tags.
    • The comments approved by the owner are displayed.
    • The user can add a comment to the photo, but it won't be displayed until the owner will approve it.
  • When displayed for owner:
    • A list of buttons will be provided:
      • Edit: redirects the user to the editPage.php to edit the current photo.
      • Add to folder: a popup form will display all the albums the user created such that one can be picked in order to add the current photo to it.
      • Delete: used when the user wants to delete the photo.
      • Add tags: a popup form will appear allowing the user to add a new tag to the photo.
      • Make public/private: Will show the opposite of the visibility status in order to change it to that.
      • Inspect: redirects the user to the inspect.php where the user can tag objects directly on the photo.
    • The owner will be able to see all comments added, and accept or reject other user's comments.

5.3.6. Albums

  • The user can see its albums and add a new album on the albums.php page.
  • When the user clicks on the Add album button, a form popup will allow the user to add an album by providing a name.
  • When clicking on one of the albums, the user will be redirected to the album.php page of the album where all of the photos that have that album in the albums list will be displayed.

5.3.7. Inspect Photo

On the inspect.php page the user can:

  • See the tags added on specific objects/areas on the photo when clicking the Show tags button.
  • Then the user can hide the tags through the Hide tags button.
  • Add a new tag by clicking an area on the photo:
    • if the user confirms (Confirm button) the coordinates of the area clicked, then a text area will appear.
    • the user can write a name for the tag and by clicking the Add tag button the tag will be added on the photo.

5.3.8. Edit Photo

On the editPage.php page the user can edit the current photo by applying CSS filters.

  • The filters available for the filter property will be provided for the user in the form of range input.

  • When clicking the Upload button, the server will update the edits in the database (default is none);
  • The button Reset is used when the user whats to eliminate the edits and display the photo in its original form.

5.3.9. Comments

CommentsInfo.php class:

  • Contains:
    • filename (the photo to which the comment was added);
    • from (the poster of the comment);
    • to (the owner of the photo);
    • status (approved/unapproved(default));
    • visibility (public or private(default));
    • text (the content of the comment);
  • Purpose:
    • To easly store the information about a comment extracted from the database;
    • Used for displaying comments.
